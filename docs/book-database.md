# Book Database V1

## 文档目标
本文件是书单推荐系统的静态书库。V1 阶段先使用 Markdown 维护书目，后续可由 Codex 转换为 `src/data/books.ts` 或数据库表。本文件兼容 `product-brief.md`、`tag-system.md`、`question-bank.md`、`personality-engine.md`、`life-stage-engine.md` 和 `recommendation-engine.md`。

## 使用原则
每本书必须尽量包含以下字段：`book_id`、`title`、`author`、`category`、`personalities`、`life_stages`、`current_challenges`、`tags`、`difficulty`、`reason`。推荐系统优先使用 `current_challenges` 和 `life_stages` 筛选书籍，再使用 `personalities` 和 `tags` 排序。`reason` 可作为结果页推荐理由的基础文案，也可由 AI 在后续版本中改写。后续扩展书目时，直接在「书目表」末尾新增一行，并确保 `book_id` 不重复，建议格式为 `BK101`、`BK102`、`BK103` 递增。

## 枚举值约束
阅读人格：地球Online升级狂、人间观察员、精神漫游者、情绪考古学家、美学收藏家、自由实验家、稳健建设者、时代FOMO怪。人生阶段：认知升级期、意义寻找期、人生重构期、创造探索期、世界观察期、稳定建设期、生活美学期。当前课题：职业发展、财富积累、创业创造、自我认同、亲密关系、世界理解、生活品质。基础标签：growth、achievement、meaning、self_awareness、aesthetics、relationship、society、future、creation、stability、wealth、freedom。

## 扩展规则
新增书籍时优先补齐 `current_challenges`、`life_stages` 和 `personalities`。如果一本书横跨多个方向，可以填写多个值，但不要过度泛化。一本书最好只绑定 1-3 个阅读人格、1-3 个人生阶段、1-3 个当前课题和 2-5 个基础标签。如果书籍信息不确定，应在 `reason` 中标记「后续需核对」，不要让不确定信息进入正式推荐结果。

## Codex 实现建议
Codex 可将本文件转换为 `src/data/books.ts`，数据结构建议如下：

```ts
export type Book = { id: string; title: string; author: string; category: string; personalities: string[]; lifeStages: string[]; currentChallenges: string[]; tags: string[]; difficulty: "入门" | "中阶" | "高阶"; reason: string; };
```

推荐引擎可按以下优先级排序：命中 `current_challenge`，命中 `primary_stage`，命中 `primary_personality`，命中 `top_tags`，控制难度组合，优先保证 6 本书中至少包含 3 本入门或中阶书，并避免同一类别过度集中。
